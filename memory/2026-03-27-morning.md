## 🧮 算法题 1：寻找两个正序数组的中位数 (LeetCode 4)

**链接**: https://leetcode.cn/problems/median-of-two-sorted-arrays/  
**难度**: Hard  
**分类**: 二分查找 / 划分数组

### 题目

> 给定两个大小分别为 `m` 和 `n` 的正序（从小到大）数组 `nums1` 和 `nums2`。请你找出并返回这两个正序数组的 **中位数** 。
> 
> 算法的时间复杂度应该为 `O(log (m+n))` 。

```
示例 1:
输入: nums1 = [1,3], nums2 = [2]
输出: 2.00000
解释: 合并数组 = [1,2,3]，中位数 2

示例 2:
输入: nums1 = [1,2], nums2 = [3,4]
输出: 2.50000
解释: 合并数组 = [1,2,3,4]，中位数 (2+3)/2 = 2.5
```

---

### 思路演进

**暴力解：合并数组**

```java
public double findMedianSortedArrays(int[] nums1, int[] nums2) {
    int m = nums1.length, n = nums2.length;
    int[] merged = new int[m + n];
    System.arraycopy(nums1, 0, merged, 0, m);
    System.arraycopy(nums2, 0, merged, m, n);
    Arrays.sort(merged);  // O((m+n)log(m+n)) 不满足要求
    int len = merged.length;
    if (len % 2 == 0) {
        return (merged[len/2 - 1] + merged[len/2]) / 2.0;
    }
    return merged[len/2];
}
```
❌ 时间复杂度 O((m+n)log(m+n))，不满足 O(log(m+n)) 要求

**优化解：二分划分数组** ⭐面试要求

核心思想：找到一条分割线，将两个数组分成左右两部分，满足：
- 左边元素个数 = 右边元素个数（或左边多1个）
- 左边所有元素 ≤ 右边所有元素

```
nums1: [1, 2, 3, 4, | 5, 6]  (分割线在第4个后)
nums2: [1, | 2, 3, 4, 5]     (分割线在第1个后)
       ↓  ↓
      left      right
```

---

### 完整代码

#### 解法：二分划分数组（O(log(min(m,n)))）

```java
class Solution {
    public double findMedianSortedArrays(int[] nums1, int[] nums2) {
        // 确保 nums1 是较短的数组，在较短的数组上二分
        if (nums1.length > nums2.length) {
            return findMedianSortedArrays(nums2, nums1);
        }
        
        int m = nums1.length, n = nums2.length;
        int left = 0, right = m;
        
        while (left <= right) {
            // nums1 的分割位置
            int partition1 = (left + right) / 2;
            // nums2 的分割位置（保证两边元素个数相等或左边多1）
            int partition2 = (m + n + 1) / 2 - partition1;
            
            // 分割线左右的值
            int maxLeft1 = (partition1 == 0) ? Integer.MIN_VALUE : nums1[partition1 - 1];
            int minRight1 = (partition1 == m) ? Integer.MAX_VALUE : nums1[partition1];
            
            int maxLeft2 = (partition2 == 0) ? Integer.MIN_VALUE : nums2[partition2 - 1];
            int minRight2 = (partition2 == n) ? Integer.MAX_VALUE : nums2[partition2];
            
            // 找到正确的分割
            if (maxLeft1 <= minRight2 && maxLeft2 <= minRight1) {
                // 总长度奇数，取左边最大值
                if ((m + n) % 2 == 1) {
                    return Math.max(maxLeft1, maxLeft2);
                }
                // 总长度偶数，取左边最大值和右边最小值的平均
                return (Math.max(maxLeft1, maxLeft2) + Math.min(minRight1, minRight2)) / 2.0;
            } else if (maxLeft1 > minRight2) {
                // partition1 太大，左移
                right = partition1 - 1;
            } else {
                // partition1 太小，右移
                left = partition1 + 1;
            }
        }
        return 0.0; // 不会执行到这里
    }
}
```

**关键点**：
1. 在较短的数组上二分，保证 O(log(min(m,n)))
2. `partition2` 的计算：`left总长度 = (m+n+1)/2 - left1长度`

### 面试追问

> **Q：为什么是 log(min(m,n)) 而不是 log(m+n)？**
> 
> A：我们在较短的数组上二分，所以时间复杂度取决于较短数组的长度。空间复杂度 O(1) 因为我们只使用了常数个变量。

> **Q：如果两个数组有重复元素怎么办？**
> 
> A：不影响算法正确性，二分判断用的是 `<=` 和 `>=`，重复元素自然会被正确处理。

## 🧮 算法题 2：盛最多水的容器 (LeetCode 11) — 热题100

**链接**: https://leetcode.cn/problems/container-with-most-water/  
**难度**: Medium  
**分类**: 双指针

### 题目

> 给定一个长度为 `n` 的整数数组 `height` 。有 `n` 条垂线，第 `i` 条线的两个端点是 `(i, 0)` 和 `(i, height[i])` 。
> 
> 找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。
> 
> 返回容器可以储存的最大水量。

```
输入: height = [1,8,6,2,5,4,8,3,7]
输出: 49
解释: 选择第1条线(高8)和第8条线(高7)，距离7，min(8,7)*7=49
```

---

### 最优解：双指针向中间收敛

**核心思想**：左右指针从两端开始，每次移动较矮的那边（因为面积 = 底 × 高，矮的高度决定了面积上限）

```java
class Solution {
    public int maxArea(int[] height) {
        int left = 0, right = height.length - 1;
        int maxArea = 0;
        
        while (left < right) {
            // 当前面积
            int w = right - left;
            int h = Math.min(height[left], height[right]);
            maxArea = Math.max(maxArea, w * h);
            
            // 移动较矮的那边（贪心：保留可能更高的边）
            if (height[left] < height[right]) {
                left++;
            } else {
                right--;
            }
        }
        return maxArea;
    }
}
```

**为什么移动较矮的？**
```
假设 height[left] < height[right]
面积 = (right-left) * height[left]

如果移动 right：
新面积 = (right-1-left) * min(height[left], height[right-1])
底边减小了，而 height[left] 没变高，面积必然不会变大

所以移动较矮的，才可能找到更高的边获得更大面积
```

**复杂度**：
- 时间：O(n) — 最多遍历 n 次
- 空间：O(1) — 只使用了几个变量

### 一致性哈希详解（缓存场景必备）

```
普通哈希: hash(key) % N
问题: 当N变化时，几乎所有key的映射都会改变 → 缓存雪崩

一致性哈希: 把节点和key都映射到环上
特点: 增删节点只影响相邻区间，大部分key位置不变
```

```java
// 一致性哈希简单实现（带虚拟节点）
public class ConsistentHash<T> {
    private final SortedMap<Integer, T> circle = new TreeMap<>();
    private final int replicas; // 虚拟节点数
    
    public ConsistentHash(int replicas, Collection<T> nodes) {
        this.replicas = replicas;
        for (T node : nodes) {
            add(node);
        }
    }
    
    public void add(T node) {
        for (int i = 0; i < replicas; i++) {
            // 每个真实节点创建多个虚拟节点
            int hash = hash(node.toString() + i);
            circle.put(hash, node);
        }
    }
    
    public T get(Object key) {
        if (circle.isEmpty()) return null;
        int hash = hash(key.toString());
        // 找环上第一个 >= hash 的节点
        if (!circle.containsKey(hash)) {
            SortedMap<Integer, T> tailMap = circle.tailMap(hash);
            hash = tailMap.isEmpty() ? circle.firstKey() : tailMap.firstKey();
        }
        return circle.get(hash);
    }
    
    private int hash(String key) {
        // FNV-1a hash or MD5
        return key.hashCode() & 0x7FFFFFFF;
    }
}
```

### 面试高频问题

> **Q：一致性哈希的虚拟节点有什么作用？**
> 
> A：1) 解决数据倾斜问题，让节点分布更均匀；2) 如果一个节点宕机，流量会分散到多个其他节点，避免单点压力突增。一般给每个真实节点创建100-200个虚拟节点。

> **Q：如果后端服务挂掉，负载均衡器如何感应？**
> 
> A：健康检查机制。1) 主动探测：定期发HTTP/TCP探测；2) 被动探测：记录请求成功率/响应时间，低于阈值标记为不健康；3) 熔断：连续失败一定次数后暂时剔除。

## 🤖 AI 大模型：模型量化与压缩

### 概念解析

大模型越来越"胖"，部署推理成本太高。**模型量化** = 用更少的比特表示权重和激活值。

```
原始模型: FP32 (32位浮点) → 每个参数 4 字节
量化后:  INT8 (8位整型)   → 每个参数 1 字节 (4x压缩)
         INT4 (4位整型)   → 每个参数 0.5 字节 (8x压缩)
```

| 量化类型 | 位宽 | 压缩比 | 速度提升 | 精度损失 |
|----------|------|--------|----------|----------|
| FP16 | 16 | 2x | 1.5-2x | 可忽略 |
| INT8 | 8 | 4x | 2-4x | 0.5-1% |
| INT4/INT3 | 4-3 | 8-11x | 3-5x | 1-3% |
| 二值化 | 1 | 32x | 5-10x | 大幅提升 |

---

### 主流量化方法

**GPTQ** — 训练后量化（Post-Training Quantization）
```
原理: 逐层优化，最小化量化误差
优势: 无需重新训练，速度快
适用: 大多数 LLM（Llama、GPT 等）
```

**AWQ** — 激活感知权重量化
```
原理: 根据激活值重要性加权，保护重要权重
优势: 相同位数下精度更高（INT4 = GPTQ INT3）
适用: 消费级 GPU 部署
```

**GGUF** — llama.cpp 专用格式
```
原理: 针对 CPU 推理优化，支持多种量化位宽
优势: 可在 Mac/低配机器上运行大模型
格式: Q4_0, Q5_K_M, Q8_0 等
```
