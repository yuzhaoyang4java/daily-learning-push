# 内容格式规范

## 推送状态文件结构

`memory/push-state.json`：

```json
{
  "lastPushDate": "2026-03-26",
  "totalDays": 3,
  "nextIndex": {
    "algorithm": 3,
    "architecture": 3,
    "ai": 3
  }
}
```

字段说明：
- `lastPushDate`：最后一次推送的日期（YYYY-MM-DD）
- `totalDays`：累计推送天数
- `nextIndex`：下一次推送各方向的题目/主题序号（0-based）

---

## 内容文件命名

`memory/YYYY-MM-DD.md`，例如 `memory/2026-03-26.md`

## 内容质量 Checklist

推送前自查：

- [ ] 算法题有至少 2 种解法的完整代码
- [ ] 架构部分有 ASCII 流程图或架构图
- [ ] 架构部分有完整 Java 代码示例（含注释）
- [ ] AI 部分有完整工作流程图
- [ ] 三个方向都有"小红书/业务场景"联想
- [ ] 三个方向都有面试相关问题
- [ ] 内容保存到 `memory/YYYY-MM-DD.md`
- [ ] 更新 `memory/push-state.json`
